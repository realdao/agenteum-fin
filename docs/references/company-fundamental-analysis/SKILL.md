# 公司基本面分析

当需要对个股进行全方位定性研究时，使用本技能。

## 分析框架

1. **公司定位**：产业链角色、产品组合、业务重心。
2. **业务拆分**：按产品/行业/地区拆分收入、成本、利润与毛利率。
3. **收入驱动**：销量、价格、结构、客户；优先使用脚本已有字段，缺失项明确标注。
4. **利润驱动**：毛利率、净利率、ROE / ROA、DuPont 拆解、估值、市场预期。
5. **增长**：营收增速、净利润增速、扣非净利润增速。
6. **运营**：存货周转/天数、应收账款天数、现金转化信号。
7. **偿债能力**：资产负债率、流动比率、速动比率、有息负债、现金覆盖倍数。
8. **商业模式**：轻/重资产、资本开支强度、经营现金流质量。
9. **竞争与护城河**：品牌、转换成本、独特资产、成本优势、议价能力。
10. **市场关注与预期差**：市场当前交易逻辑、担忧因素、潜在催化。
11. **跟踪指标**：下一期财报及未来 3 个月可观测指标。
12. **资产负债表清洁度**：主业聚焦程度、债务简洁性、金融资产规模、非经常性损益、投资收益、公允价值变动、商誉、无关投资。

## 财务质量评分（F-Score）

本技能将 Piotroski F-Score 模型作为基本面分析的**定量组成部分**，基于脚本输出的结构化数据进行打分。

**评分维度与 9 条规则**：

- **盈利能力（4 分）**
  1. 扣非 ROA > 0
  2. 经营现金流 > 0
  3. 扣非 ROA 较上一年度改善
  4. 经营现金流 > 净利润

- **杠杆与流动性（3 分）**
  5. 非流动负债 / 总资产 同比下降
  6. 流动比率同比改善
  7. 期末股本未增加（无增发）

- **运营效率（2 分）**
  8. 毛利率同比改善
  9. 总资产周转率同比改善

**评分区间**：
- 0–3 分：财务质量较弱 / 高风险
- 4–6 分：财务质量中性
- 7–9 分：财务质量较强

## 脚本调用

```powershell
# 基本面结构化数据
python .\skills\company-fundamental-analysis\scripts\company_fundamental_data.py --code 000681 --format json
python .\skills\company-fundamental-analysis\scripts\company_fundamental_data.py --code 000681 --format yaml

# F-Score 财务质量评分
python .\skills\company-fundamental-analysis\scripts\f_score.py --code 000681 --format json
python .\skills\company-fundamental-analysis\scripts\f_score.py --code 000681 --period 20251231 --format yaml
```

脚本仅返回结构化数据，不做投资结论；解读由 LLM 基于上述框架单独完成。

## 数据源

- **AkShare**：财务摘要、新浪财经报表、同花顺业务简介、东方财富主营业务构成、主要股东。
- **腾讯行情 API**：当前价格、市值、PE / PB 字段。
- **本地计算**：利润率、增长、DuPont 分析、TTM 估值、应收账款天数、债务比率、资产负债表清洁度检查。

## 解读规则

- 判断主业质量时，**优先使用扣非净利润**，而非归母净利润。
- 基于非经常性损益的 PE 质量较低；分析时应同时展示归母 TTM PE 与扣非 TTM PE。
- 轻资产或服务类业务，存货指标可能不适用；不要强制套用制造业模板。
- **缺失数据应明确标注**，禁止猜测或编造。
- 以最新年报为稳定基准；以最新季报作为更新与预警信号。
- 本技能**不依赖外部网络搜索**，所有分析必须基于脚本返回的结构化数据；如需同业比较，应通过其他技能或外部研究补充，不在本技能内完成。
